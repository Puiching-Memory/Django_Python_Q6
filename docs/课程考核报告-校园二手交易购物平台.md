## 1 需求描述

本项目名称为“基于 Django 的校园二手交易购物平台设计与实现”。系统面向校园内二手物品交易场景，提供商品浏览、分类查询、商品详情查看、购物车管理、订单提交和订单提交成功展示等功能。

系统主要需求如下：

1. 商品展示页面展示商品分类以及最新商品。
2. 商品分类展示页面展示某一类商品的全部商品信息。
3. 商品详情页面展示单个商品的详细信息。
4. 购物车页面展示用户加入购物车的商品，统计商品金额，并支持删除购物车物品。
5. 订单提交页面展示已选商品列表、订单信息填写区和支付方式选择区。
6. 订单提交成功页面展示订单商品信息、订单信息和商品金额统计。
7. 系统提供后台管理、示例商品数据、同类商品推荐和订单编号生成等自定义功能。

## 2 系统架构说明

系统采用 Django MTV 架构实现。

Model 层负责数据库模型设计，包含商品分类、商品、订单和订单明细四类数据。View 层负责业务处理，包括商品展示、分类筛选、详情展示、购物车操作、订单提交和订单成功展示。Template 层负责页面展示，使用 Django 模板语法渲染数据。购物车使用 Django session 保存，适合课程作业中的未登录用户购物流程。

项目主要目录如下：

```text
CampusSecondhandShop/   项目配置、全局路由、WSGI/ASGI
Shop/                   购物平台应用
templates/Shop/         六个功能页面模板
static/css/             页面样式
static/images/          本地商品图片资源
docs/                   课程文档与报告
```

## 3 数据库设计说明

系统使用 SQLite 数据库，便于本地运行和提交。数据库包含以下主要数据表：

| 数据表    | 说明       | 主要字段                                                                                 |
| --------- | ---------- | ---------------------------------------------------------------------------------------- |
| Category  | 商品分类表 | Name、Slug、Description、SortOrder                                                       |
| Product   | 商品表     | Category、Name、SellerName、Condition、Price、Stock、ImageName、Description、CreatedAt   |
| Order     | 订单表     | OrderNumber、ReceiverName、Phone、Address、PaymentMethod、Remark、TotalAmount、CreatedAt |
| OrderItem | 订单明细表 | Order、Product、ProductName、UnitPrice、Quantity                                         |

表关系说明：

1. 一个商品分类可以包含多个商品，Category 与 Product 是一对多关系。
2. 一个订单可以包含多个订单明细，Order 与 OrderItem 是一对多关系。
3. 订单明细可关联商品，同时保存下单时的商品名称、单价和数量，便于订单历史信息展示。

## 4 系统设计与实现

首页商品展示由 `ProductListView` 实现，读取所有商品分类和最新商品，并渲染 `ProductList.html`。页面顶部提供分类导航，用户可以进入不同分类页面，也可以直接把商品加入购物车。

分类展示由 `CategoryDetailView` 实现，根据分类 Slug 查询分类信息和该分类下全部商品，并渲染 `CategoryDetail.html`。

商品详情由 `ProductDetailView` 实现，根据商品 ID 查询商品详细信息，展示价格、成色、库存、卖家、发布时间和商品描述，并提供加入购物车表单。

购物车功能使用 session 实现。`CartAddView` 用于加入购物车，`CartView` 用于展示购物车商品和金额合计，`CartRemoveView` 用于删除购物车商品。

订单提交由 `CheckoutView` 实现。页面展示购物车中的已选商品列表、订单信息表单和支付方式。用户提交后，系统生成订单编号，保存订单和订单明细，并清空购物车。

订单提交成功页面由 `OrderSuccessView` 实现。页面展示订单编号、商品明细、收货信息、支付方式、提交时间和订单总金额。

## 6 心得体会或建议

通过本次课程大作业，我进一步理解了 Django 项目的基本开发流程，包括项目创建、应用划分、模型设计、路由配置、视图函数编写、模板渲染和静态资源管理。购物平台虽然功能相对基础，但完整覆盖了从商品浏览到购物车再到订单提交的业务流程，能够帮助我把 Python 语法、Web 框架和数据库操作结合起来。
